**Key Technical Decisions from CONTEXT.md:**
- Library: pdf-parse (simpler API, good for table extraction, Node.js compatible)
- Table structure: Auto-detect structure from PDF (don't hardcode positions)
- Fallback method: Yes, multiple strategies (try next method if one fails)
- Data source priority: PDF is primary, IDX API is exploratory

**Data Format (sample PDF available):**
- File: ownership_one_percent_202603.pdf (test this during implementation)
- Expected columns: Rank, Stock Code, Holder Name, Shares, Percentage
- Multi-page tables with consistent structure

**Critical Understanding:**
- IDX PDFs only contain holders with >1% ownership (sum will NOT equal 100%)
- This is expected behavior, not an error

<task type="auto">
  <name>Task 2: Create PDF parsing utilities</name>
  <files>src/lib/utils/pdf-utils.ts</files>
  <action>
Create utility functions for PDF data parsing:

**detectTableStructure(text: string): TableStructure**
- Analyze raw PDF text to identify table headers
- Detect column positions and delimiters
- Return structure: { headers: string[], startLine: number, delimiter: string }
- Try multiple patterns: pipe-separated, whitespace, tab-aligned

**parseStockCode(text: string): string | null**
- Validate IDX stock code format (exactly 4 uppercase letters)
- Return null if invalid
- Pattern: /^[A-Z]{4}$/

**parsePercentage(text: string): number | null**
- Parse percentage from various formats: "5.23%", "5.23", "5,23"
- Handle Indonesian decimal comma (convert to point)
- Return null if not a valid percentage (0-100)

**parseShares(text: string): number | null**
- Parse share count (remove commas, dots as thousand separators)
- Handle Indonesian format: "1.000.000.000" → 1000000000
- Return null if not a valid number

**cleanHolderName(name: string): string**
- Trim whitespace
- Remove extra spaces between words
- Normalize common variations (PT vs Pt, Tbk vs TBK)
- Return cleaned name

Each function must handle edge cases and return null for invalid inputs.

**tryPrimaryStrategy(pdfData: any, structure: TableStructure): ExtractedOwnershipRecord[]**
Primary extraction strategy:
1. Split PDF text into lines
2. Skip header rows until table data starts
3. Parse each data row using parseStockCode, parsePercentage, parseShares
4. Extract stock code, holder name, shares, percentage
5. Assign rank based on position in table
6. Return array of records

**detectPeriodFromFileName(fileName: string): string**
Extract period from filename format: "ownership_one_percent_YYYYMM.pdf"
Return "YYYY-MM" format (e.g., "2026-03")

Handle:
- Multi-page PDFs (continue parsing across pages)
- Page headers/footers (detect and skip)
- Missing values (skip row or mark as null)
- Indonesian number formats (commas as decimals)

Use pdf-parse library (not pdf.js or pdf2json).

  <name>Task 4: Implement fallback extraction strategies</name>
  <files>src/lib/services/pdf-extractor.ts</files>
  <action>
Add fallback strategies to pdf-extractor.ts:

**tryFallbackStrategy1(pdfData: any): ExtractedOwnershipRecord[]**
Fallback 1: Regex-based extraction
- Use regex patterns to find stock codes and associated data
- Pattern: /([A-Z]{4})\s+(.+?)\s+([\d.,]+)\s+([\d.,]+%)/
- Extract data from anywhere in PDF text
- Less strict about table structure

**tryFallbackStrategy2(pdfData: any): ExtractedOwnershipRecord[]**
Fallback 2: Line-by-line scanning
- Scan all lines for anything looking like a stock code
- Extract following words as holder name
- Look ahead for percentage and share values
- Useful for malformed table structures

**tryAllStrategies(filePath: string): Promise<PDFExtractionResult>**
Updated main function:
1. Try primary strategy
2. If returns < 10 records, try fallback 1
3. If still < 10 records, try fallback 2
4. Use best result (most records)
5. Log which strategy succeeded in metadata

Each fallback must be independent and handle complete extraction.

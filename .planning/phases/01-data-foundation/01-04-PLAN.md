- Update workflow: API endpoint for manual triggering
- Automation: Manual only (no cron jobs for now)
- Recovery method: Rollback on error (all-or-nothing transaction)
- Idempotency: Yes, idempotent (re-running same PDF is safe — skip or update existing)
- Raw data backup: Always backup raw PDFs and API responses
- Testing approach: Direct to production (no separate test environment)
- Severity levels: Critical = stop, Error = rollback, Warning = continue

**Error Handling Requirements:**
- Transaction rollback on critical errors
- Idempotent operations (upsert, not insert)
- Severity levels determine behavior
- Raw PDFs backed up before processing

- Tables: periods, emiten, holders, ownership_records

- Foreign keys enforce referential integrity

  <action>
Create data access layer using Drizzle ORM:

**upsertPeriod(data: ExtractedPeriod): Promise<Period>**
Insert or update period record:
- Check if period ID exists
- If exists: update pdfFileName, extractionVersion
- If not: insert new record
- Return period object

**upsertEmiten(code: string, name: string): Promise<Emiten>**
Insert or update emiten record:
- Use stock code as ID
- Upsert name and sector (null for now)
- Return emiten object

**upsertHolder(name: string, type: string): Promise<Holder>**
Insert or update holder record:
- Normalize name using canonical name rules
- Check if similar holder exists (fuzzy match)
- If exists: return existing holder
- If not: insert new holder with canonicalName, originalName, type
- Return holder object

**upsertOwnershipRecord(record: ExtractedOwnershipRecord, periodId: string, emitenId: string, holderId: number): Promise<OwnershipRecord>**
Insert or update ownership record:
- Use composite key (periodId, emitenId, holderId)
- Upsert rank, sharesOwned, ownershipPercentage
- Return ownership record object

**findPeriodById(periodId: string): Promise<Period | null>**
Check if period already exists:
- Query periods table by ID
- Return period or null

**withTransaction<T>(callback: (tx: Transaction) => Promise<T>): Promise<T>**
Execute callback within database transaction:
- Begin transaction
- Execute callback
- Commit on success
- Rollback on error
- Return result

Use Drizzle's insert().onConflictDoUpdate() for upserts.

<task type="auto">
  <name>Task 2: Create import service with transaction handling</name>
  <files>src/lib/services/import-service.ts</files>
  <action>
Create main import orchestrator service:

**backupRawPDF(filePath: string): Promise<string>**
Backup raw PDF file:
- Copy PDF to backups/ directory
- Add timestamp to filename: ownership_one_percent_202603_20260314_123456.pdf
- Return backup file path
- Create backups/ directory if not exists

**importFromPDF(filePath: string, options?: ImportOptions): Promise<ImportResult>**
Main import workflow:
1. Backup raw PDF using backupRawPDF()
2. Extract data using extractFromPDF(filePath)
3. Validate result using validateExtractionResult()
4. If validation has critical errors: throw error (stop import)
5. Start database transaction using withTransaction()
6. Inside transaction:
   a. Upsert period record
   b. For each record:
      - Upsert emiten
      - Upsert holder
      - Upsert ownership record
   c. Check for validation warnings (log but continue)
7. Commit transaction
8. Return ImportResult with { success, recordsImported, warnings, errors }

**importFromJSON(jsonData: string): Promise<ImportResult>**
Import from pre-extracted JSON:
- Parse JSON string
- Validate using validateExtractionResult()
- Same workflow as importFromPDF() without extraction step

**checkIdempotency(periodId: string): Promise<boolean>**
Check if period already imported:
- Query database for period
- Return true if exists, false if not

**ImportOptions** type:
- skipValidation: boolean (default: false)
- forceUpdate: boolean (default: false) — update existing records
- dryRun: boolean (default: false) — don't actually insert

**ImportResult** type:
- success: boolean
- recordsImported: number
- periodId: string
- warnings: string[]
- errors: string[]
- extractionTime: number (ms)

Handle all errors and rollback transaction on failure.
  </action>

  <action>
Create Next.js API route for PDF upload and import:

**POST /api/import**
Handle multipart form data:
1. Parse uploaded PDF file from formData
2. Validate file type (application/pdf)
3. Validate file size (max 50MB)
4. Call importFromPDF() with uploaded file path
5. Return JSON response with import result

Request format:
- Content-Type: multipart/form-data
- Field: file (PDF file)

Response format (success):
```json
{
  "success": true,
  "recordsImported": 500,
  "periodId": "2026-03",
  "warnings": ["Ownership sum: 87.3%"],
  "errors": [],
  "extractionTime": 2345
}
```

Response format (error):
```json
{
  "success": false,
  "error": "Invalid PDF format",
  "details": "..."
}
```

Error handling:
- 400: Invalid file type or size
- 413: File too large
- 422: Validation errors
- 500: Server error (extraction or database failure)

Add rate limiting (max 10 imports per hour per IP).

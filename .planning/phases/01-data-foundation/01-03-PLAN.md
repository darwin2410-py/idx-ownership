<objective>
Implement data quality validation using Zod schemas to ensure extracted IDX ownership data integrity before database insertion.

Purpose: Catch missing, malformed, or invalid data early in the pipeline while logging warnings for expected anomalies (like < 100% sum), preventing corrupt data from entering the database.

Output: Working validation service with Zod schemas that checks data quality and logs issues without stopping the import process.
</objective>

**Key Technical Decisions from CONTEXT.md:**
- Validation library: Zod (runtime type validation)
- Validation timing: Validate after extraction (separate step for cleaner separation)
- Validation strictness: Warnings only — ownership sum will NOT equal 100% because data only includes >1% holders
- Severity levels: Critical = stop entire import, Warning = log but continue

**Critical Understanding from CONTEXT.md:**
- IDX PDFs only contain holders with >1% ownership
- The sum of percentages will NOT equal 100% (smaller holders excluded)
- This is EXPECTED behavior — should log WARNING, not ERROR

**Validation Requirements:**
- Stock codes must match IDX format (4 uppercase letters)
- Percentages must be between 0 and 100
- Shares must be positive integers
- Holder names cannot be empty
- Rank must be positive integer
</context>

<task type="auto">
  <name>Task 3: Implement validation service</name>
  <files>src/lib/validation/ownership-validator.ts</files>
  <action>
Create main validation service:

**validateRecord(record: ExtractedOwnershipRecord): ValidationResult**
Validates a single ownership record:
1. Parse and validate stock code using stockCodeSchema
2. Validate holder name using holderNameSchema
3. Parse and validate shares using sharesSchema
4. Parse and validate percentage using percentageSchema
5. Validate rank using rankSchema
6. Return result with errors/warnings

**validateOwnershipPercentage(records: ExtractedOwnershipRecord[]): ValidationWarning[]
Check if ownership percentage sum is reasonable:
1. Sum all ownership percentages
2. If sum > 100%, add WARNING (possible data error)
3. If sum < 80%, add INFO warning (expected — only >1% holders shown)
4. If 80% <= sum <= 100%, no warning
5. Return array of warnings

**validateExtractionResult(result: PDFExtractionResult): ValidationResult**
Validate complete extraction result:
1. Check if records array exists and has > 0 items
2. Validate each record using validateRecord()
3. Run validateOwnershipPercentage() on all records
4. Check for duplicate (stock, holder) combinations
5. Check rank sequence (should be 1, 2, 3... without gaps)
6. Return complete ValidationResult with summary

**buildValidationSummary(results: ValidationResult[]): DataQualityMetrics**
Aggregate metrics from multiple validation runs:
- Total rows, valid rows, error rows
- Average extraction time
- Warning count
- Ownership sum
- Return DataQualityMetrics object

Handle Zod parsing errors and convert to ValidationError format.
